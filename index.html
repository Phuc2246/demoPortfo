Digital Instrument Cluster & VCU

Automotive Dashboard System on Embedded Linux

Raspberry Pi 4 STM32 Qt/QML CAN Bus Yocto C++
View on GitHub

A complete automotive instrument cluster system built with Embedded Linux on Raspberry Pi 4, featuring a real-time Qt/QML HMI communicating with a STM32-based VCU over CAN Bus. Custom Yocto image was built with optimized boot time.

What I did

  • Built custom Embedded Linux distribution using Yocto for Raspberry Pi 4 with Qt runtime, SocketCAN, and systemd integration
  • Designed real-time HMI dashboard with Qt/QML — animated gauges, speed display, warning indicators
  • Developed multi-threaded C++ backend for SocketCAN communication between VCU and cluster
  • Programmed STM32F103 VCU firmware with JSON-based I/O pin mapping for flexible configuration
  • Optimized Linux boot sequence reducing startup time significantly

Bare-metal Dino Game Controller

STM32 Firmware without HAL Library

STM32F407 Bare-metal C UART EXTI ISR Timer/PWM Python
View on GitHub

A bare-metal embedded project demonstrating deep understanding of microcontroller internals. The STM32F407 acts as a physical game controller for Chrome's Dino game, with all drivers written from scratch using direct register access — no HAL, no CMSIS drivers.

What I did

  • Wrote GPIO, USART, and Timer drivers from scratch using direct register manipulation on STM32F407
  • Implemented EXTI hardware interrupt handlers with software debouncing for reliable input detection
  • Built bidirectional UART communication protocol to control a Python/Pygame Dino game on PC
  • Configured Advanced Timers and PWM for dynamic LED visual feedback effects
  • Achieved <5ms input response latency with 150KB total firmware footprint
